Everything you need to vibe code

Select, capture, modify — all from your running web app.

🎯

Element Selection

Click any element in your app to capture its DOM structure, computed styles, and surrounding HTML. The LLM knows exactly what to modify.

📸

Screenshot Capture

Take screenshots of the page or specific elements. Vision-capable models (GPT-5.4, Claude Opus 4.6, Gemini 3.1 Pro) use this to understand layout.

🌐

Network & Console Logs

Automatically captures fetch/XHR requests and console output. The LLM understands your API responses, errors, and app state.

🧠

Thinking Models

Automatic reasoning config per model. GPT-5.4 uses reasoning_effort, Claude uses extended thinking, Gemini uses thinking_level — all pre-configured.

📂

Multi-Repo Context

Add your backend alongside your frontend. The LLM can read and modify files across multiple repositories simultaneously.

⚡

Instant Hot Reload

AI modifies your source files directly. Your dev server's HMR picks up changes automatically — see updates in milliseconds.

Three steps to vibe coding

1

Start your dev server, then run OpenMagic

OpenMagic runs a reverse proxy that wraps your existing dev server. Zero code changes. Works with any framework.

$ npx openmagic --port 3000
2

Select your LLM provider and model

Pick from 50+ pre-configured models across 9 providers. Just select from the dropdown and paste your API key. One-time setup.

3

Select elements and describe changes

Click any element, take a screenshot, or just describe what you want. The AI proposes code changes, you approve, and the UI updates instantly.
